# Struct-ML Project

Experiments on **heterogeneous relational GNNs** using [RelBench](https://relbench.stanford.edu/) datasets.

---

## Environment Setup

A single conda environment (`structml1`) is shared across all aspects of the project.

### Create from scratch

```bash
conda env create -f environment.yml
conda activate structml1
```

`torch-scatter` and `torch-sparse` are fetched from PyG's pre-built CUDA wheel
server via `--find-links` in the yml. If conda's pip runner still fails on them
(some older conda versions ignore `--find-links`), install them manually after
creating the env:

```bash
conda activate structml1
pip install torch-scatter torch-sparse \
    -f https://data.pyg.org/whl/torch-2.4.1+cu121.html
```

### Update an existing environment

```bash
conda env update -f environment.yml --prune
conda activate structml1
```

### Verify the install

```bash
python -c "import torch, torch_geometric, torch_frame, relbench, wandb; print('OK')"
python -c "import torch; print('CUDA:', torch.cuda.is_available())"
```

> **CUDA version:** The yml targets PyTorch 2.4.1 + CUDA 12.1 (`+cu121`).
> For a different CUDA version replace `+cu121` everywhere in `environment.yml`
> (e.g. `+cu118`) and update the `--find-links` URL to match.

### wandb

Log in once before running any training:
```bash
wandb login          # prompts for API key
# or: export WANDB_API_KEY=<your-key>
```

## Project Structure

```
structML_project/
├── environment.yml      ← shared conda environment (all aspects)
├── README.md            ← this file
└── aspect1/             ← GNN directionality experiment
    ├── README.md        ← how to run aspect1
    ├── preprocess.py
    ├── models.py
    ├── train.py
    ├── sweep.yaml       ← wandb sweep definition (162 runs)
    ├── run_sweep.sh     ← SLURM array launcher
    ├── generate_configs.py
    ├── processed/       ← generated by preprocess.py
    └── results/         ← metrics.csv written by train.py
```
